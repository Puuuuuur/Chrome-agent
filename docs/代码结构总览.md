# 浏览器 Agent 交付物总览

这个文档面向交付物使用者，目标是快速看懂：

- 交付物里有哪些主要目录
- 请求是怎么从入口一路走到底层执行的
- 哪些文件属于业务能力，哪些属于技术底座

## 目录角色

- `app.py`
  - 服务入口
  - 注册页面、聊天接口、skills 接口、固定业务接口

- `对话智能体.py`
  - 页面渲染层
  - 负责内置聊天界面和状态展示

- `智能体调度.py`
  - 同步调用入口
  - 把 Flask / 页面调用转成 skills 异步执行

- `chat_cli.py`
  - 终端交互入口
  - 通过本地 HTTP 接口与浏览器 Agent 持续对话

- `chat_memory/`
  - 会话记忆目录
  - `models.py`
    - 会话上下文、消息记录、语义命中结构
  - `postgres_store.py`
    - PostgreSQL 会话/消息持久化
  - `service.py`
    - 记忆加载、摘要、写回编排

- `rag_kb/`
  - 本地 RAG 知识库目录
  - `models.py`
    - 知识块命中结构
  - `milvus_store.py`
    - PDF chunk 的 Milvus 存储
  - `service.py`
    - PDF 扫描、文本切分、向量索引、检索

- `rag_store/`
  - 本地知识源目录
  - `source_pdfs/`
    - 放待索引 PDF

- `skills/`
  - 业务 skills 目录
  - `base.py`
    - 共享基类、schema 校验、公共辅助函数
  - `registry.py`
    - skill 注册、意图分发、统一生命周期
  - `skill_browser_react.py`
    - 通用浏览器 skill
  - `skill_creditchina_query.py`
    - 对外交付的稳定信用中国查询 skill

- `tools/`
  - 技术底座目录
  - `tool_browser_runtime.py`
    - 浏览器会话管理、页面诊断、creditchina 查询执行、结果落盘、工具暴露
  - `tool_captcha.py`
    - 验证码生成与识别
  - `tool_model_client.py`
    - OpenAI client 与 API Key 读取

- `智能体配置.py`
  - 环境变量与默认配置

- `docs/信用中国查询业务逻辑说明.md`
  - 业务文档
  - 重点讲用户输入、意图识别、执行链路和结果输出

- `docs/requirements.txt`
  - 依赖清单

- `../浏览器agent生成文件/信用中国查询/`
  - 默认结果目录

- `.session/`
  - 会话态目录

## 用户入口

- 页面：`/playwright-agent/`
- 聊天接口：`/api/playwright-agent/chat`
- 显式 skill 列表：`/api/playwright-agent/skills`
- 显式 skill 执行：`/api/playwright-agent/skills/run`
- 固定业务接口：`/api/creditchina/query`
- 终端入口：`python3 chat_cli.py`

## 主链路

1. 请求先进入 `app.py`
2. 页面请求交给 `对话智能体.py`
3. 接口请求交给 `智能体调度.py`
4. `智能体调度.py` 再调用 `skills/registry.py`
5. `skills/registry.py` 完成：
   - 意图分发
   - 会话记忆加载
   - 本地 RAG 检索
   - 输入校验
   - 浏览器生命周期
   - skill 执行
   - 结果写回 PostgreSQL
6. 具体 skill 再决定执行方式：
   - `browser_react`：开放式浏览器 ReAct 流程
   - `creditchina_query`：固定业务查询流程
7. 最底层浏览器与查询执行都落在 `tools/tool_browser_runtime.py`

## 当前结论

- 对外交付的稳定业务查询能力是 `creditchina_query`
- `private-api` 现在只是 `creditchina_query` 内部优先使用的实现策略，不再是单独公开 skill
- 技术底座已经拆进 `tools/`，业务 skills 已拆进 `skills/`
- 会话记忆已独立拆进 `chat_memory/`，不混入浏览器执行底座
- 本地 PDF RAG 已独立拆进 `rag_kb/`，Milvus 主要承担知识库检索
