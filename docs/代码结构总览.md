# 浏览器 Agent 代码结构总览

这套代码现在按“入口 -> 页面/UI -> 调度 -> 浏览器工具 -> 支撑模块 -> 文档/产物”来理解最顺。

## 目录角色

- `app.py`
  - 独立 Flask 服务入口。
  - 暴露 `/playwright-agent/`、`/api/playwright-agent/chat`、`/api/creditchina/query`。

- `对话智能体.py`
  - 页面渲染层。
  - 负责把“已就绪/未就绪”、默认配置、聊天 UI 拼出来。
  - 对外导出 `invoke_playwright_agent`、`invoke_creditchina_query` 等正式入口。

- `智能体调度.py`
  - 智能体主调度层。
  - 负责创建顶层 agent、组织消息、决定工具调用、汇总最终返回。
  - 如果你想看“用户一句话是怎么变成浏览器动作的”，从这里开始最合适。

- `agent工具.py`
  - 浏览器执行层 + 工具层。
  - 这里最重，包含：
  - Playwright 会话管理
  - CDP/本地浏览器探测
  - 页面截图、挑战页判断、结果落盘
  - 提供给 agent 的可调用工具

- `模型工具.py`
  - 模型客户端和 API Key 读取。
  - 只做一件事：给其它模块稳定提供 OpenAI client。

- `验证码工具.py`
  - 图形验证码处理。
  - 包含验证码生成、图像预处理、本地识别/OCR 调用逻辑。

- `智能体配置.py`
  - 所有默认配置集中地。
  - 包括模型、浏览器模式、CDP 地址、会话目录、产物目录等。

- `docs/信用中国_API化改造方案.md`
  - 业务背景和接口设计文档。
  - 适合在看懂代码主链之后，再回头理解为什么做成现在这样。

- `../浏览器agent生成文件/信用中国查询/`
  - 当前默认的结果与调试产物主目录。
  - 会按“日期/单次运行目录”保存 `json/txt/html/png`。

- `.session/`
  - 会话态和 cookies。
  - 当前默认不再把结果文件写到这里；现存 `.session/results/` 属于旧结构遗留。

- `.artifacts/`
  - 历史或显式自定义的调试目录。
  - 如果没有覆盖 `PLAYWRIGHT_AGENT_ARTIFACT_DIR`，当前截图、HTML 和结果文件默认都会落到 `../浏览器agent生成文件/信用中国查询/`。

## 代码主链

1. 请求从 `app.py` 进入。
2. 页面请求走 `对话智能体.py` 渲染 UI。
3. 聊天/API 请求进入 `智能体调度.py`。
4. `智能体调度.py` 调用 `agent工具.py` 里暴露的浏览器工具。
5. `agent工具.py` 再依赖：
   - `模型工具.py` 处理模型 client
   - `验证码工具.py` 处理验证码
   - `智能体配置.py` 读取默认配置

## 如果你只想快速读懂

- 想看服务入口：`app.py`
- 想看页面长什么样：`对话智能体.py`
- 想看 agent 怎么思考：`智能体调度.py`
- 想看浏览器到底怎么被控制：`agent工具.py`
- 想看验证码怎么处理：`验证码工具.py`
- 想看环境变量都有哪些：`智能体配置.py`

## 当前建议

- 第一次读，不要先扎进 `agent工具.py` 全文。
- 先顺着 `app.py -> 对话智能体.py -> 智能体调度.py` 把主流程串起来。
- 再回头按功能去啃 `agent工具.py`，会轻松很多。
